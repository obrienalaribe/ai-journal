# AI Automation Strategy, Built in Public

I'm OB — a software engineer building AI agent systems that do real work, not demos.

This journal documents the weekly reality of running **OBOS**, a production dashboard powered by Claude Code agents. 11 dashboard pages, 44+ API endpoints, 17 RSS feeds, DeFi monitoring, Schengen compliance tooling — all built and maintained by human-AI collaboration.

## What I Build

**OBOS Dashboard** is my operating system for life as a location-independent engineer. It automates:

- **News aggregation** — 17 RSS sources filtered to 45 articles/week using bucketed keyword scoring + LLM analysis
- **DeFi monitoring** — 13 parallel data scripts, multi-source OHLCV charts, Aave yield tracking
- **Task management** — GitHub-integrated board with live agent tracking and PR preview deploys
- **Schengen compliance** — 90/180-day rolling window calculator with timezone-safe date math
- **Content pipeline** — This blog, generated from real session data and published weekly

The full codebase is open source. Every commit, every Living Rule, every friction analysis — all public.

## Why This Journal Exists

After analyzing 79 Claude Code sessions (54 hours of compute), I found a **24.4% friction rate** — nearly a quarter of agent time was wasted on late verification, planning rabbit holes, and auth failures.

So I built systems to fix it: post-edit type checking hooks, mandatory plan files, a 20-question self-reflection protocol, and 18+ Living Rules that survive context compaction.

Each week I share what worked, what broke, and what I changed. Real data, real failures, real fixes.

## The Stack

- React 18 + TypeScript + Vite (frontend)
- Express server, JSON filesystem (no database)
- Claude Code agents with structured skills and hooks
- GitHub Pages + Jekyll (this blog)
- Doppler for secrets, PM2 for process management
