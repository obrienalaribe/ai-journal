# Filesystem-backed dashboards need auto-generation, not just cron

## What Happened
A 24/7 financial dashboard showed no data for the current day because daily snapshots were generated by a nightly cron job at 22:00. The fix was startup-time generation + 4-hour periodic refresh using the same API-only script (zero LLM cost, ~30s runtime).

## The Insight
When an endpoint reads from dated filesystem snapshots, cron-only generation creates a blind spot between midnight and the next cron run. If the generation script is cheap (pure API calls), run it on server startup and periodically — don't wait for cron.

## By The Numbers
- 0 → 6 snapshots per day (startup + every 4h)
- 18 → 20 E2E tests for DeFi Monitor

## Pattern
Startup-check + periodic refresh for filesystem-backed data endpoints (guard flag + age check + `setInterval`)
